# 集成天文学和物理学可视化模块

## Why
现有的天文学和物理学可视化模块功能完整但与主页导航系统隔离，用户无法从平台主页直接访问这些优秀的学科可视化内容。集成这些模块将提供统一的用户体验，让用户能够在一个平台中访问所有学科的可视化功能。

## 概述
将现有的天文学可视化模块 (`ai_visual_astronomy`) 和物理学可视化模块 (`ai_visual_physics`) 集成到万物可视化平台的主页导航系统中，使用户能够从主页直接访问这些模块。

## 背景
项目已经拥有完整的天文学和物理学可视化模块，但它们位于独立目录中，没有集成到主页的导航系统。用户无法从主页直接访问这些优秀的可视化内容。

## 解决方案
通过以下步骤将模块集成到主应用中：
1. 在主应用的modules目录下创建astronomy和physics子目录
2. 复制现有模块内容到新位置
3. 更新主页的URL映射配置
4. 验证所有模块的访问和导航功能

## 技术细节
- **天文学模块位置**: `/ai_visual_astronomy/frontend/index.html`
- **物理学模块位置**: `/ai_visual_physics/physics-visualization/web/index.html`
- **目标位置**: `/main-app/modules/astronomy/` 和 `/main-app/modules/physics/`
- **URL映射**: 使用相对路径确保跨模块导航正常工作

## 验收标准
- 用户可以从主页点击天文学和物理学模块按钮
- 所有模块都能正常加载和导航
- 保持现有模块的独立性和功能完整性
- 统一的用户体验和界面风格
