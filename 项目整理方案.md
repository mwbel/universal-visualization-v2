# 万物可视化项目结构整理方案

## 📋 整理目标
将项目目录按照版本和功能进行分类，便于维护和开发。

## 🗂️ 建议的目录结构

### 当前V2版本（保留在根目录）
- `backend-v2/` - 最新后端系统
- `frontend-v2/` - 最新前端系统
- `start-v2.sh` - V2启动脚本
- `README-v2.0.md` - V2项目文档
- `openspec/` - 项目规范文档

### 归档到 archive/old-versions/
- `main-app/` - 旧版主应用
- `backend-env/` - 旧版后端环境
- `backend-performance/` - 性能测试版本
- `universal-visualization-v2-clean/` - 清理版备份（已包含备份文件）

### 归档到 archive/tests/
- `custom-viz-demo.html` - 自定义可视化演示
- `debug-test.html` - 调试测试页面
- `main-app-fixed.html` - 修复版主应用测试
- `main-app-test.html` - 主应用测试
- `minimal-test.html` - 最小测试页面
- `simple-test.html` - 简单测试页面
- `standalone-test.html` - 独立测试页面
- `test-extended-distributions.html` - 扩展分布测试
- `test-fixed-axes.html` - 固定坐标轴测试
- `test-visualization.html` - 可视化测试

### 保留在根目录的配置文件
- `package.json` - Node.js配置
- `package-lock.json` - 依赖锁定文件
- `.gitignore` - Git忽略文件
- `.editorconfig` - 编辑器配置
- `.nvmrc` - Node版本配置
- `CLAUDE.md` - AI助手指令

### 已整理的目录（保持现状）
- `参考/` - 参考资料
- `学习文档/` - 学习资料
- `之前版本/` - 历史版本

### 其他文件处理建议
- `START_HERE.html` - 可移动到tests或保留
- `quick-start.sh` - 保留（快速启动脚本）
- `start-dev-server.js` - 保留（开发服务器）
- `stop_servers.sh` - 保留（停止脚本）
- `node_modules/` - 保留（依赖包）

## ✅ 整理后的好处
1. **清晰的项目结构** - V2版本文件集中在根目录
2. **便于维护** - 旧版本和测试文件归档管理
3. **减少混乱** - 避免多版本文件混杂
4. **提高效率** - 快速找到需要的文件

## 🔧 执行建议
1. 先创建backup目录进行备份
2. 逐步移动文件到对应归档目录
3. 更新相关配置文件中的路径引用
4. 测试V2系统功能正常
