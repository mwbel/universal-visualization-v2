# 万物可视化 - 技术小白AI开发学习指南

## 🎯 学习目标
通过修复和优化万物可视化项目，学会使用AI工具进行开发，理解前后端交互，掌握基本调试技能。

## 📚 学习路径（渐进式）

### 阶段一：理解系统架构（当前阶段）

#### 1.1 系统概览
- **前端**：HTML/CSS/JavaScript，用户界面
- **后端**：Python FastAPI，处理业务逻辑
- **AI集成**：通过API调用AI服务生成可视化

#### 1.2 发现的问题
从刚才的测试中，我们发现了以下问题：
1. ❌ 模板列表API返回格式不正确
2. ❌ 模板渲染失败（模板不存在：default）
3. ✅ 基础功能正常

### 阶段二：逐步修复问题

#### 步骤1：检查模板文件
让我们先看看模板文件是否存在：

```bash
# 查看后端目录结构
ls -la backend-v2/

# 查看模板目录
find . -name "*.html" -type f
```

#### 步骤2：理解模板系统
模板系统的结构：
- `templates/` - 存放HTML模板
- `agents/` - 各学科的处理逻辑
- `static/` - 静态文件

### 阶段三：创建第一个简单模板

#### 3.1 创建基础模板
让我们创建一个简单的默认模板：

```html
<!DOCTYPE html>
<html>
<head>
    <title>万物可视化 - {{title}}</title>
</head>
<body>
    <h1>{{title}}</h1>
    <div id="visualization">
        {{content | safe}}
    </div>
</body>
</html>
```

#### 3.2 理解模板变量
- `{{title}}` - 标题变量
- `{{content | safe}}` - 内容变量（允许HTML）

### 阶段四：调试API问题

#### 4.1 查看API返回格式
使用curl命令查看API返回：

```bash
curl http://localhost:8000/api/v2/templates
```

#### 4.2 修复数据格式
检查返回的是字符串还是JSON对象

### 阶段五：前端交互

#### 5.1 理解前端代码结构
- `frontend-v2/js/app.js` - 主应用逻辑
- `frontend-v2/js/api-service.js` - API调用
- `frontend-v2/index.html` - 主页面

#### 5.2 添加调试日志
在前端代码中添加console.log来调试
